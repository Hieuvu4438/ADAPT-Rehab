\section{Conclusion}
\label{sec:conclusion}

We presented \system{}, a multimodal AI rehabilitation system designed for elderly users. The system integrates real-time whole-body 3D pose estimation via RTMW3D-L (133 keypoints covering body, hands, and face) at 129.7~FPS, multi-indicator facial state detection with OpenFace~3.0, voice-interactive LLM coaching, and a clinically-inspired six-dimension scoring framework.

The key contributions are: (1)~to our knowledge, the first integrated rehabilitation system combining whole-body 3D pose estimation, multi-indicator facial state detection, voice-interactive LLM coaching, and clinically-inspired kinematic scoring in a single real-time framework; (2)~a whole-body 3D pose estimation pipeline using RTMW3D-L from MMPose, sustaining 129.7~FPS with full 133-keypoint coverage on a single consumer GPU; (3)~a multi-indicator facial state detection pipeline integrating PSPI-inspired pain estimation, PERCLOS-based fatigue monitoring, blink/yawn detection, and engagement analysis; and (4)~a cross-skeleton evaluation framework that exercises mapping across MediaPipe, SMPL-24, and Kinect~v2 formats.

This paper is a feasibility study: the pipeline is computationally practical, but clinical validation with elderly participants, ground-truth pose-accuracy benchmarking, and scoring calibration against clinician ratings are deferred to future work. The scoring framework analysis in Section~\ref{sec:experiments} provides a candid baseline: the feature representation carries exercise-discriminative signal (separation ratio 1.40), but the aggregate score and clinical-rating correlation ($r = 0.045$) remain unresolved, defining clear calibration targets.

Future work includes: (1)~ground-truth evaluation on public rehabilitation benchmarks (UI-PRMD, KIMORE) using the cross-skeleton mapping framework; (2)~clinical user studies with elderly participants to assess usability and therapeutic effectiveness; (3)~metric-scale calibration to enable direct comparison with clinical goniometer measurements; (4)~edge-based LLM deployment to eliminate internet connectivity requirements; and (5)~multi-camera fusion to address self-occlusion limitations.
